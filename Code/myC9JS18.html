18. Unobtrusive JavaScript

This is based on www.webstandards.org; which is appears stopped active work on 2010, and then becomes an education organization instead. Their courses are worth chekcing out

It says that there are 4 benefits if using Unobstrusive Javascript techniques:

  1. Usability: It should just work without user's attention
  2. Graceful Degradation: If it doesn't work, it should just fail silently and go away, without user attention or impact to other contents of the page
  3. Accessibility: If js script failed, it shouldn't stop delivering other HTML contents, although without those add-ons from the js file
  4. Separation: For future or other developers, all js scripts &. resources should be maintained separately, without impact or stop other js scripts' work

Three techniques will be shown here: Separation of Behavior from Markup, Namespace(a.k.a "Global Abatement"), Degrading Gracefully.

  • Separation of Behavior from Markup: Separate HTML Content from Js Behavior -- Already done using "<script type="text/javascript" src="myscript18.js"></script>"
  • Namespace: to avoid conflicts with other Js scripts, library or other programs, try not to make too much global functions or global variables. Just put them inside one global object, and call the attributes of that object for your method or data usage. So by putting all your code under this global object, you effectively reduce your footprint under global scope. BTW, there is another method called "Modular Pattern", which is more popular but not talked about here (more advanced technique you should search for yourself). This is very necessary when you are working on a larger project that requires more than one programmers.
  • Degrading Gracefully: see Js file for details
